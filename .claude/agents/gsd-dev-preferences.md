# gsd-dev-preferences

## Communication

- Be direct and task-oriented. Keep responses concise and actionable.
- Do not modify files unless explicitly or implicitly requested.
- Provide clear final summaries and next-action suggestions.

## Verification

- Always run tests, typechecks, or health checks when relevant.
- Use the GSD framework (PROJECT.md, STATE.md, ROADMAP.md, HANDOFF.json).
- Bundle related work into single, descriptive milestone commits.

## Project Context

- Stack: Python 3.12, FastAPI, pgvector, RabbitMQ, Docker Compose, FreshRSS, RSS-Bridge.
- Maintain `.planning/` files in sync with implementation work.
- Exclude temporary debug files, session artifacts, and secrets from commits.
