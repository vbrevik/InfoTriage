## Acceptance Criteria

Verdict is **GO** if **all** are true. **PARTIAL** if mechanism GO but exactly one of (b)/(c) fails.
**NO-GO** if mechanism fails or two of (b)/(c) fail.

| # | Criterion | Test |
|---|-----------|------|
| a | **Mechanism**: pgvector-style cosine similarity correctly orders mE5-large L2-normalized mean-pooled entity vectors (sanity: synthetic SHuffled random GAussian fails this test — a real mE5 come-out of XLM-RoBERTa large must show >0 for same-entity pairs). | Document the minimum / maximum / p10 / p90 of pairwise cosine on the cross-language same-entity set + the distinct-entity set. Both must be >0 collapse fraction of same-entity vectors above some threshold T_max-distinct. |
| b | **Cross-language same-entity merge**: at the chosen threshold T*, every same-entity cross-language pair has cosine ≥ T* AND version coverage AT LEAST one entity spans ≥ 2 distinct `lang` values among the validation forms. | The chosen T* from default corpus must satisfy `min(same_pairs.cosine) ≥ T*` AND the corpus includes at least one entity with `{en, no, ru}` or `{en, ru}` or `{no, ru}` cross-language surface forms. |
| c | **Distinct-entity separation**: at T*, every pair from the distinct_pairs corpus has cosine strictly < T*. (`max(distinct_pairs.cosine) < T*.`) | The chosen T* must satisfy `max(distinct_pairs.cosine) < T*`. |

**Threshold sweep table** (mandatory output, mirrors R2 format):

- For each T in {0.75, 0.76, ..., 0.98}:
  - `collapse_rate` = fraction of same_pairs whose cosine ≥ T
  - `control_overmerge` = count of distinct_pairs whose cosine ≥ T
- Choose T* = smallest T where `collapse_rate == 1.0` AND `control_overmerge == 0` (ALL same-merge, zero over-merge), if such a T exists. Otherwise T* = first T where both next-best bars are meet. Otherwise T* = 0.85 (no clear separation, NO-GO sub-verdict).

## Constraints (Hard)

1. **No cloud endpoints.** All embeddings from local `intfloat/multilingual-e5-large` model dir at
   `~/.omlx/models/multilingual-e5-large/` OR HF cache at `~/.cache/huggingface/hub/models--intfloat--multilingual-e5-large/`.
2. **mE5-large protocol**: 1024-dim, **mean-pooled** (NOT CLS-pooled — mE5 documentation says mean),
   L2-normalized, `query: ` prefix for query-side embeddings (asymmetric retrieval).
3. **`utf-8` corpus**: surface forms include Cyrillic (НАТО, Россия, Зеленский) explicitly.
4. **No silent elevation.** PARTIAL stays PARTIAL per `SPIKE-FINDINGS.md` constraints.
5. **Threshold authoritative only for entity-level linking.** Item-level dedup threshold (R2 mE5-large
   @ 0.84) is a different problem and stays at 0.84. Do not conflate.

## Risks / Watch-outs

- **Existing synthetic-vectors verdict**: Be sure the script does NOT regress to synthetic-fallback.
  `--allow-synthetic` MUST default to False in any new offline-mode. Document this in code.
- **Mean pooling vs CLS pooling**: mE5 explicitly requires mean pooling with attention mask.
  Sentence-transformers library handles this; if we ever drop to manual code we MUST mirror their
  attention-masked mean-pool implementation.
- **Threshold 0.85 vs 0.84 confusion**: R2 used 0.84 for **item-level dedup** (article-article similarity).
  R3 wanted 0.85 for **entity-level linking** (entity surface form to entity surface form). These are
  different metrics at different points in the pipeline. 999.3's output is the entity-level value.
  Phase 8 plan should harmonize.
