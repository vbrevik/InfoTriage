## Objective

Build the missing Obsidian vault-writer that projects high-value enrichment items and the SAB into Obsidian `.md` files. This satisfies ROADMAP SC2 and unlocks SC3 (email surfacing in Obsidian).

## Context

- VERIFICATION.md found that no `vault_writer.py` exists.
- The vault-writer should emit one `.md` file per high-value enrichment item (score >= 8, or all items with CCIR).
- Front-matter should follow existing codec pattern from `libs/contracts/src/contracts/_codec.py`.
- Body contains item summary and other fields.
- `[[Entity]]` wikilinks are generated via a lightweight heuristic (e.g., proper-noun extraction, known topic list). This is an interim solution until Phase 8 entity resolution.
- Email-sourced items (`imap://`) must appear in the vault.
- Also write a projection of the SAB into Obsidian (e.g., `vault/obsidian-sab.md`).

## Scope

- Create `apps/brief/vault_writer.py` with functions:
  - `write_item_obsidian(item: dict, vault_path: Path, config: dict | None = None) -> Path`
  - `write_sab_obsidian(enrichment_rows: list[dict], vault_path: Path) -> Path`
  - `extract_entities(text: str, known_topics: list[str] | None = None) -> list[str]`
  - `render_wikilinked(text: str, entities: list[str]) -> str`
- Integrate into consumer's post-rendering workflow.
- Add tests in `tests/test_vault_writer.py`.
- Add `data/obsidian/` to `.gitignore` if not already.
- Use environment variables:
  - `INFOTRIAGE_VAULT_PATH` defaults to `data/obsidian`
  - `VAULT_INCLUDE_EMAIL` defaults to `1` (include imap:// items)

## Template Design

### File naming
- Item files: `vault/{item_id}.md` (use sanitized title slug if present)
- SAB file: `vault/obsidian-sab.md`

### Front-matter fields (YAML)
- `title`
- `date` (ISO 8601)
- `ccir`
- `score`
- `cnr`
- `bucket`
- `source`
- `url`
- `item_id`

### Body format
```md
## Summary
{summary}

## Why
{why}

## Topics
{wikilinked topics}
```

### Wikilink heuristic
- Use a simple regex to find capitalized proper nouns and phrases.
- Optionally use a small known-topic list (e.g., CCIR topics).
- Example: `[[Norway]]`, `[[NATO]]`, `[[Ukraine]]`.

## Tasks

### Task 1: Create `apps/brief/vault_writer.py` — core functions

**Read first:**
- `libs/contracts/src/contracts/_codec.py` — front-matter codec pattern.
- `apps/brief/renderer.py` — enrichment row structure.
- `apps/triage/digest.py` — for entity extraction ideas.

**Action:**
Implement:
1. `write_item_obsidian(item: dict, vault_path: Path, config: dict | None = None) -> Path`
   - Writes a YAML front-matter block + body to `vault_path/{item_id}.md`.
2. `write_sab_obsidian(enrichment_rows: list[dict], vault_path: Path) -> Path`
   - Projects the SAB (or a summary of items) into `vault/obsidian-sab.md`.
3. `extract_entities(text: str, known_topics: list[str] | None = None) -> list[str]`
   - Simple proper-noun extractor using regex and a small known-topic list.
4. `render_wikilinked(text: str, entities: list[str]) -> str`
   - Replaces entities with `[[Entity]]` marks.

**Use existing front-matter codec** — follow the pattern in `_codec.py` for serialization.

**Acceptance criteria:**
- `write_item_obsidian` creates a valid Obsidian `.md` file with front-matter and body.
- `extract_entities` returns a list of unique extracted entities.
- `render_wikilinked` replaces each entity with `[[Entity]]`.
