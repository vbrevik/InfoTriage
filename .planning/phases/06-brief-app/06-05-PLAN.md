<objective>
Close the Phase-6 UAT gap where db_live test fixtures pointed at the PRODUCTION Postgres
(host port 22000) and TRUNCATEd every infotriage table on each run — the direct cause of
"SAB shows no items" (the DB had been wiped by a prior pytest run). Make it structurally
impossible for a test to touch production, and add a guard so it cannot silently return.

probe, so any `pytest` run with prod reachable wipes live articles/enrichments/embeddings.

Output: Three reworked test files (explicit INFOTRIAGE_TEST_DSN, no prod fallback), a
throwaway test-Postgres compose file so db_live tests remain runnable, and an always-run
regression guard that fails if any test file targets the prod host port.
</objective>

In all three test files, remove every hardcoded production DSN and every hardcoded prod-port
reachability probe. The tests must resolve their DSN from a NEW, test-only environment
variable INFOTRIAGE_TEST_DSN — never from INFOTRIAGE_PG_DSN (that is the running app's DSN and
in dev/compose it resolves to production) and never from a hardcoded literal.

Concrete edits (apply the equivalent change in each of the three files; match each file's
existing structure and style — do not reformat surrounding code):

1. Delete the module-level prod DSN constant (`DEV_DSN = "postgresql://...localhost:22000/infotriage"`
   in test_store_integration.py and test_triage_enrichment.py; the inline
   `os.environ.get("INFOTRIAGE_PG_DSN", "postgresql://...22000/infotriage")` fallback inside the
   postgres fixture branch in test_store_contract.py). Introduce a shared constant naming the
   env var, e.g. `TEST_DSN_ENV = "INFOTRIAGE_TEST_DSN"`.

2. Replace `_get_dsn()` so it returns `os.environ.get(TEST_DSN_ENV)` with NO fallback (return
   value may be None). In test_store_contract.py, replace the inline fallback expression the same
   way (read INFOTRIAGE_TEST_DSN only).

3. Replace the reachability helper (`_pg_reachable()`) so it derives host+port from
   INFOTRIAGE_TEST_DSN instead of the hardcoded `("localhost", <prod port>)` tuple:
     - If INFOTRIAGE_TEST_DSN is unset/empty -> return False (this is what makes db_live tests
       auto-skip when no test DB is configured, preserving R8's skip-when-unavailable behavior).
     - Otherwise parse host/port via `psycopg.conninfo.conninfo_to_dict(dsn)` (psycopg is already
       a dependency), defaulting host to "localhost" and port to 5432 when absent, then attempt
       `socket.create_connection((host, int(port)), timeout=1.0)` and return True/False.
   Rename it to `_test_db_reachable()` for clarity and update all references; keep `_PG_UP`-style
   module-level evaluation semantics (evaluated once at collection time).

4. Update the skipif reason strings from the old ":22000 unreachable" wording to
   "INFOTRIAGE_TEST_DSN unset or test DB unreachable — db_live test skipped". Keep BOTH marks
   (the registered `db_live` marker AND the skipif) exactly as the current dual-mark pattern does,
   so `pytest -m db_live` selection and auto-skip both keep working.

5. The `_truncate_all(dsn)` helpers and the fixtures stay as-is functionally, but they now receive
   the INFOTRIAGE_TEST_DSN value. Because the fixtures/tests are guarded by the reachability
   skipif, `_get_dsn()` is only ever called when a test DSN is configured, so it is non-None at
   the call sites. Do NOT weaken TRUNCATE — the whole point is that it now runs against an
   isolated test DB, never prod.

6. Scrub prose references to the prod port in module docstrings/comments in these three files
   (e.g. "against Postgres :<prod port>", "auto-skipped when :<prod port> unreachable"): rephrase
   to reference INFOTRIAGE_TEST_DSN and the test DB. This keeps the regression guard (Task 2) and
   these files consistent.

Do NOT touch scripts/shadow_run.py (it deliberately targets the app DSN and is out of scope),
and do NOT refactor test bodies beyond the DSN/reachability wiring above.
