<objective>
Wave 1 foundation: define the `SabPublished` event schema in contracts, and implement the
core markdown renderer (`renderer.py`) that produces `brief.md`, `cluster.md`, `list.md`, and
`bluf.md` from enrichment rows in Postgres. This is data-layer work — no event wiring, no
HTTP serving, no Docker yet.

Purpose: deliver the rendering logic that `worker.py`'s SAB path depends on. The renderer
adapts `digest.py`'s `write_brief()` / `write_list()` / `write_bluf()` functions to read
from enrichment rows instead of Fever verdict dicts.

Output: 1 new contract event (SabPublished), 1 new module (apps/brief/renderer.py), contract
tests exercising both inmemory and postgres stores.
</objective>
