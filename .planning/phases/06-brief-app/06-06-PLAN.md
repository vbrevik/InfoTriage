<objective>
Eliminate the second half of the Phase-6 UAT gap: the triage worker (and any store read caller)
left a psycopg connection "idle in transaction" — observed 8.5 hours — which let a queued
fixture TRUNCATE (AccessExclusive) pile locks behind it and blocked all reads, so /sab could
not read the tables.

Purpose: PostgresStore's read methods (get_item, list_items, get_enrichment, find_near_duplicate)
run a SELECT on the non-autocommit connection and never commit or roll back, so the implicit
read transaction stays open until the next write. The brief app already ends its read txns
explicitly (apps/brief/main.py, apps/brief/consumer.py end each read txn to avoid
idle-in-transaction); the store's own read methods — the path the triage worker and digest use —
do not. Close that path at the source, and add a conservative server-side backstop so any future
leak (for example a raw cursor() reader) self-heals instead of wedging the database.

Output: read methods that leave the connection IDLE after returning, a connection-level
idle-in-transaction timeout backstop, and a db_live regression test proving it.
</objective>

Make the four read methods leave the connection IDLE, and add a conservative server-side backstop.

1. In each read method, after the SELECT result is fetched and BEFORE returning, end the implicit
   read transaction by calling self._conn.rollback(). rollback (not commit) is correct here: these
   are pure reads with nothing to persist, so rollback simply closes the read-only transaction and
   releases its snapshot/locks without risk of committing a stray write. Apply to:
     - get_item: after fetchone(), before both the `return None` (miss) and the `return Item(...)`
       (hit) paths — ensure the rollback runs on BOTH branches (e.g. capture the row, rollback,
       then branch). Never leave a return path that skips ending the txn.
     - list_items: after fetchall() in BOTH the source_type_in branch and the else branch, before
       building/returning the list.
     - get_enrichment: after fetchone(), before both the miss and hit return paths.
     - find_near_duplicate: after fetchone(), before all return paths (the near-duplicate hit and
       the None result).
   Add a short inline comment on each — end read txn — avoid idle-in-transaction — matching the
   established convention already used in apps/brief/main.py and apps/brief/consumer.py.
   Do NOT change any write method (put_item, put_enrichment, put_embedding, put_blob): they must
   keep their explicit commit() so DD-5 write+audit atomicity is preserved. Do NOT switch the
   connection to autocommit (that would break put_item's two-statement single-transaction guarantee
   and the aborted-transaction semantics that test_put_item_failure_raises depends on).

   backstop, not the primary fix.
