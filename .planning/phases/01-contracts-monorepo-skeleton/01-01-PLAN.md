<objective>
Create `libs/contracts` — a pip-installable Python package that is the single source of
truth all InfoTriage apps depend on: the pydantic v2 `Item` model, the four bus event
schemas, a lossless PyYAML frontmatter<->payload codec, and a `typing.Protocol` bus-client
interface with a working in-memory implementation. Wire it for local dev via an editable
install and prove every behavior with a new `tests/test_contracts.py`.

Purpose: Establishes the shared contract seam (SPEC R1-R4) that the monorepo restructure
(plan 01-02) imports from, and that every future app (Phase 2+) pip-installs. No pipeline
behavior changes here — this plan only adds the new package and its tests.

Output: `libs/contracts/**`, `tests/test_contracts.py`, `requirements-dev.txt`, and an
editable install of the `contracts` package in the active environment.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| markdown/frontmatter -> codec | Untrusted Obsidian YAML frontmatter is parsed into a Python dict by from_frontmatter. |
| caller -> Item/event constructors | Arbitrary caller-supplied fields cross into pydantic validation. |
| dependency supply chain | pydantic / PyYAML / pytest are pulled into the runtime via the editable install. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Tampering | _codec.from_frontmatter (YAML parse) | high | mitigate | Parse only with yaml.safe_load (never the unsafe/full loader); safe_load disables arbitrary Python object construction, blocking YAML-deserialization code execution. Enforced by acceptance grep in Task 2. |
| T-01-02 | Information Disclosure | libs/contracts package contents | high | mitigate | No secrets/tokens/endpoints in the package (SPEC P2, D-02); all config from .env at runtime. Verified by the package-grep acceptance and the prohibitions block. |
| T-01-03 | Tampering | Item.payload open dict | low | accept | payload is intentionally an open dict for Phase 5 extensibility; validation of payload contents is Phase 5's responsibility, not the contract layer. |
| T-01-04 | Spoofing | Item.id sha256 dedup key | low | accept | SHA-256 collision resistance is industry standard; acceptable for content-dedup use. |
| T-01-SC | Tampering | pydantic / PyYAML / pytest installs | low | accept | No new downloads — all three are already installed and audited (01-RESEARCH.md §Package Legitimacy Audit: Approved; the SUS verdict is a documented PyPI-stats false positive). No [ASSUMED]/[SUS]/[SLOP] package requires a blocking checkpoint this plan. |
</threat_model>
