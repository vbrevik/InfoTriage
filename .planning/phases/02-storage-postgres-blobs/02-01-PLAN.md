    001-schema.sql: `CREATE SCHEMA IF NOT EXISTS infotriage`, then `CREATE EXTENSION IF NOT EXISTS vector` (DD-1 realizes the namespace as a real Postgres schema, lowercase/unquoted). This file must sort first so the schema and extension exist before later files reference `infotriage.*`.

    002-articles.sql: `infotriage.articles` with hybrid mapping (D-02) — real columns `id TEXT PRIMARY KEY` (the contracts.Item sha256 computed id), `source`, `source_type`, `url` (NOT NULL DEFAULT ''), `title`, `ts TIMESTAMPTZ`, `lang`, `summary`, `body_ref` (sha256 → data/blobs), `payload JSONB NOT NULL DEFAULT '{}'`, `created_at`. Add btree indexes on source_type, ts DESC, lang. Do NOT add a tsvector/GIN column (D-02a — FTS deferred to the search/RAG phase).

    003-vectors.sql: `infotriage.entities` (id SERIAL PK, name, name_norm, lang, type, `embedding vector(1024)`), `infotriage.entity_links` (id SERIAL PK, entity_id FK→entities, item_id FK→articles, mention, lang), `infotriage.embeddings` (id SERIAL PK, item_id FK→articles, `embedding vector(1024) NOT NULL`, model, created_at). The 1024 dimension is the locked embedder contract (D-05a, mE5-large). Create HNSW indexes on entities.embedding and embeddings.embedding `USING hnsw (embedding vector_cosine_ops) WITH (m=16, ef_construction=64)` (D-05b). Use plain `CREATE INDEX IF NOT EXISTS` (never CONCURRENTLY — it cannot run in a transaction; RESEARCH Pitfall 4).

    004-audit.sql: `infotriage.audit` recording store write events (DD-5) — id BIGSERIAL PK, op TEXT (e.g. put_item / put_blob), table_name, item_id, ts TIMESTAMPTZ DEFAULT NOW().

    005-stubs.sql: `infotriage.enrichment` and `infotriage.ccir` as minimal forward-declared stubs (DD-3) — id SERIAL PK, item_id FK→articles, created_at — each with a SQL comment noting that the scoring phase (≈P4) and P5/P8 own the real columns. These remain bare so later phases extend them.

    All FKs reference `infotriage.articles(id)` / `infotriage.entities(id)` with fully-qualified names. Use parameterless DDL only (no `%s`) so psycopg3 runs each file as a text string.
