<objective>
Build `libs/ingest_common`, the shared adapter toolkit every Phase 4 ingest container depends on. It contains the three pieces that would otherwise be copy-pasted four times: the idempotent persist+publish helper (the get_item pre-check pattern from RESEARCH Finding 2), the FastAPI `POST /run` single-instance-lock trigger app factory (D-01/D-02), and the env-driven store/bus construction helpers. This plan also lands the two unit tests that pin the two behaviors most likely to break — idempotency (R6) and the concurrency lock (D-01) — and registers the package + the five not-yet-created adapter directories on the pytest path so Wave 2 plans add zero shared-file edits.

Purpose: a single, tested normalization backbone so the four adapter plans (Wave 2) only have to supply source-specific fetch logic and a one-line `main.py`.
Output: installed `ingest_common` package, two green unit tests, updated root `pyproject.toml` pythonpath.
</objective>

    - Re-running the full adapter twice over the same single source item leaves exactly ONE entry in the store and exactly ONE payload on the "item.ingested" queue (R6).
    - Published payload is a dict with keys source, source_type, ts (ts is item.ts.isoformat()); item_id passed as item.id.
  </behavior>

Implement `async def persist_and_publish(store, bus, item) -> bool` in `persist.py` following RESEARCH Pattern 2 (per D-01/ADR-001/ADR-007): capture `existing = store.get_item(item.id)`; call `store.put_item(item)` unconditionally (upsert); if `existing is None`, `await bus.publish("item.ingested", item_id=item.id, payload={"source": item.source, "source_type": item.source_type, "ts": item.ts.isoformat()})` and return True; else return False.
