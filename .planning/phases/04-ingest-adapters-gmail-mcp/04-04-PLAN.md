<objective>
Build `ingest-obsidian` (greenfield) to read Obsidian Web Clipper markdown clips from `$OBSIDIAN_VAULT_PATH/articles-inbox/*.md`, map their frontmatter to `Item` (D-08/D-09) via the existing `libs/contracts.from_frontmatter` codec, and `persist_and_publish` to Postgres+bus. The vault is bind-mounted read-only — the adapter never writes to it (ADR-004). Poll-based: each `POST /run` re-scans the inbox directory.

Purpose: add the Obsidian clip source as a containerized Collection adapter (ADR-003), read-only (ADR-004), reusing the tested YAML codec rather than hand-rolling a parser.
Output: `apps/ingest-obsidian/` container source + Dockerfile + R4 test.
</objective>
