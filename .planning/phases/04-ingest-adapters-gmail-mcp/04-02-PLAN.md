<objective>
Containerize the multi-mailbox IMAP bridge as `ingest-imap`. Reuse the proven READ-ONLY fetch logic from `apps/ingest/imap_to_atom.py` (mailbox config loader, provider dispatch, search, MIME decode), but replace the Atom-XML output with `Item` construction → `ingest_common.persist_and_publish` (Postgres + bus). No Atom file is written — email is triage-only (SPEC R1). The adapter runs as a long-lived FastAPI server exposing the shared `POST /run` trigger.

Purpose: bring multi-mailbox IMAP (REQUIREMENTS C-13) onto the Postgres+bus architecture, read-only (ADR-004), as a containerized Collection-stage adapter (ADR-003).
Output: `apps/ingest-imap/` container source + Dockerfile + R1 test.
</objective>
