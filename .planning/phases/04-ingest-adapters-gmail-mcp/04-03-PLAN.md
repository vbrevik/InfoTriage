<objective>
Containerize the YouTube bridge as `ingest-youtube`. Reuse the proven yt-dlp listing + stub-transcription logic from `apps/ingest/yt_to_atom.py`, but produce DUAL output (SPEC R2 / CONTEXT specifics): (1) construct `Item`, write the transcript text to a content-addressed blob (`Item.body_ref`), and `persist_and_publish` to Postgres+bus; AND (2) keep writing `data/feeds/youtube-<slug>.xml` Atom so FreshRSS river-browsing still works. Transcription is forced to the stub path — `mlx-whisper` requires macOS Metal and is never installed in the Linux container.

Purpose: bring YouTube channels (REQUIREMENTS C-9) onto Postgres+bus while preserving the FreshRSS Atom surface, read-only (ADR-004), containerized Collection adapter (ADR-003).
Output: `apps/ingest-youtube/` container source + Dockerfile + R2 test (incl. blob-dedup and empty-channel backstops).
</objective>
