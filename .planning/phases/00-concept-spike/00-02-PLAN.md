<objective>
Prove the RabbitMQ broker model for InfoTriage events: a working AMQP publish->consume round-trip
across two dummy services using the designed topic exchange and routing keys, plus a poison message
that dead-letters to a DLQ. This resolves R1 and supplies the decision recorded in ADR-007.

Purpose: the bus topology (exchanges, routing keys, DLX/DLQ, acks, confirms) is chosen but unproven;
Phase 3 builds the production bus on this exact topology.
Output: r1_topology/publisher/consumer scripts + a durable R1 verdict.
</objective>
