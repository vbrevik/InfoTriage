## Data flow (today)

```
RSS / YT / Reddit ─▶ FreshRSS :8088 ─┐                                       ┌─▶ SAB / digest
websites ─ rss-bridge :3000 ─▶       ├─ Fever API ─▶ fever_triage.py ─▶ score ┤
Gmail ─ gmail_to_atom.py ─▶ feeds: ──┘                                       └─▶ mark junk read
                                                          ↑                              ↑
                                                    triage_score.py (qwen3.6 via oMLX)
                                                          ↑
                                                       ccir.md (the taxonomy)
```

One process boundary: scripts. They share a `.env` and a code-import relationship (`digest.py` reuses `fever_triage.py` and `triage_score.py`). No package, no API surface.

## What's deliberately absent

- Tests — none. Acceptance is "runs against local oMLX + FreshRSS and the verdict looks right."
- A `trimail` module — `sys.path.insert(0, …)` instead. Fine at this scale; revisit if it grows.
- Source control of `data/` — `.gitignore` excludes it. Re-provisioning instructions exist.
- Locked env schema — `.env.example` referenced in README but not present in tree. Gap to close.
