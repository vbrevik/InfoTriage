## Directory Purposes

**bridge/**
- Purpose: Adapters that convert external sources into Atom feed format.
- Contains: Three converters (Gmail, IMAP, YouTube) + shared utility (HTML escape).
- Key files: `gmail_to_atom.py` (single-account Gmail via IMAP), `imap_to_atom.py` (multi-account IMAP dispatch), `yt_to_atom.py` (YouTube with optional transcription).
- Output: Atom XML files written to `data/feeds/<name>.xml` for FreshRSS to ingest.
- Schedule: Run via cron/launchd (typically hourly or on-demand).

**score/**
- Purpose: The triage engine — LLM scoring, bucket assignment, digest generation.
- Contains: Scorer (reusable function), Fever API loop (auto-mark-read), digest generator (four narrative modes).
- Key files: `triage_score.py` (core scorer + sample demo), `fever_triage.py` (Fever loop), `digest.py` (reporting + synthesis).
- Config: Inlines `ccir.md` as the behavior definition; `LLM_BASE_URL`, `LLM_API_KEY`, `LLM_MODEL` from env.
- Behavior: Stateless (pure functions); calls external LLM endpoint (oMLX/Ollama) + FreshRSS Fever API.

**opml/**
- Purpose: Feed metadata and OPML import/export.
- Contains: `feeds.opml` (live feed list, ~45 feeds, 11 outline categories), validator script.
- Key files: `_check.py` (URL live-check + XML well-formedness).
- Usage: Import into FreshRSS via web UI → Subscription management ▸ Import.

**tests/**
- Purpose: Unit test suite; validates scorer robustness, drift guards, XML/OPML parsing.
- Contains: Six test modules (scorer, digest, OPML, bridges).
- Run: `python3 -m pytest tests/` or per-file `python3 tests/test_*.py`.
- Coverage: Error handling, edge cases, sync guards (CCIR_ORDER ↔ ccir.md).

**docs/**
- Purpose: Strategic & design documentation.
- Contains: `ARCHITECTURE.md` (ADRs, design decisions, prior-art survey, intelligence-cycle reframing, comparison to Taranis/World Monitor/TAK/Palantir).

**data/**
- Purpose: Runtime state (git-ignored by default except examples).
- Contains: Feeds (Atom XML), FreshRSS SQLite database, digest outputs, verdict log.
- **feeds/**: Atom files written by bridges; served to FreshRSS via Docker network alias `http://feeds/`.
- **freshrss/**: FreshRSS installation (volume-mounted in compose); contains SQLite database, user data, cache.
- **digests/**: Markdown digest outputs (cluster.md, brief.md, list.md, bluf.md); ephemeral, regenerated each run.
- **verdicts.jsonl**: Append-only JSON log of all scored items; survives FreshRSS wipes.

## Key File Locations

**Entry Points:**
- `bridge/gmail_to_atom.py`: Run via cron; pulls Gmail IMAP, writes `data/feeds/gmail.xml`.
- `bridge/imap_to_atom.py`: Run via cron; pulls multi-IMAP, writes `data/feeds/imap-*.xml`.
- `bridge/yt_to_atom.py`: Run via cron; pulls YouTube, writes `data/feeds/youtube-*.xml`.
- `score/triage_score.py`: CLI entry for ad-hoc scoring (stdin, --file, --sample).
- `score/fever_triage.py`: Scheduled task (cron); Fever loop with auto-mark-read.
- `score/digest.py`: Scheduled task; digest generator (cluster/brief/bluf/list modes).

**Configuration:**
- `.env`: Live secrets (FRESHRSS_FEVER_*, LLM_*, GMAIL_*, IMAP_*, YT_*).
- `.env.example`: Template; copy to .env and edit.
- `ccir.md`: Taxonomy (behavioral config); edit to change what counts as important.
- `docker-compose.yml`: Container stack (FreshRSS, feeds, rss-bridge).

**Core Logic:**
- `score/triage_score.py`: `score_item(it)` function (reused by fever_triage, digest).
- `score/fever_triage.py`: `fever()` function (Fever API client), `main()` loop.
- `score/digest.py`: `fetch_window()`, `cluster()`, `write_cluster()`, `write_brief()`, `write_bluf()`, `write_list()`.

**Data:**
- `data/feeds/`: Atom XML files (generated by bridges).
- `data/freshrss/`: FreshRSS state (SQLite + config).
- `data/digests/`: Digest markdown (ephemeral, regenerated each run).
- `data/verdicts.jsonl`: Verdict log (persistent, append-only).

**Testing:**
- `tests/test_score_parse.py`: LLM JSON error handling, fallback mechanism.
- `tests/test_write_bluf.py`: BLUF synthesis, citation formatting.
- `tests/test_ccir_sync.py`: Drift guard (CCIR_ORDER ↔ ccir.md).
- `tests/test_opml_*.py`: OPML validation and round-trip.
- `tests/test_bridge_escape.py`: HTML escape for Atom safety.

## Naming Conventions

**Files:**
- Bridge converters: `{source}_to_atom.py` (e.g., `gmail_to_atom.py`).
- Scorer/digest: `{action}.py` (e.g., `triage_score.py`, `digest.py`).
- Tests: `test_{feature}.py` (e.g., `test_score_parse.py`).
- Utilities: `_util.py` (shared, non-standalone).
- Config: `.env`, `.env.example`, `docker-compose.yml`.
- Metadata: `ccir.md`, `.gitignore`, `README.md`, `.continue-here.md`.

**Directories:**
- `bridge/`, `score/`, `opml/`, `tests/`: lowercase, plural (logical domains).
- `data/`, `docs/`: lowercase, singular (containers of instance data / documentation).
- `.planning/`: GSD-specific orchestration (phases, audit logs, codebase maps).

**Functions:**
- `score_item(it)`: Scores one item; returns enriched verdict dict.
- `fever(api_key, query, **params)`: Fever API client.
- `load_ccir()`: Load taxonomy from file.
- `load_dotenv(path)`: Load .env file into os.environ.
- `cluster(items)`: Keyword-overlap dedup clustering.
- `fetch_window(cutoff_epoch, hardcap)`: Fetch Fever items in time range.
- `write_cluster()`, `write_brief()`, `write_bluf()`, `write_list()`: Render digest modes.

**Variables:**
- Env vars: `FRESHRSS_FEVER_URL`, `LLM_BASE_URL`, `GMAIL_USER`, etc. (SCREAMING_SNAKE_CASE).
- Module constants: `CCIR_ORDER`, `CCIR_PATH`, `OSLO`, `STOP`, `SAMPLE`. (SCREAMING_SNAKE_CASE).
- Local vars: `items`, `verdicts`, `clusters`, `prompt`, etc. (snake_case).

## Where to Add New Code

**New Bridge (New Source Type):**
- File: `bridge/{source}_to_atom.py`.
- Template: Copy `bridge/gmail_to_atom.py`, adapt `main()` to fetch from your source, use `feedgen` to write Atom.
- Output: Write to `data/feeds/{source}.xml`.
- Integration: Update README.md Bridges section; add to cron schedule; optionally extend `docker-compose.yml` if the source is containerized.

**New Digest Mode:**
- File: Edit `score/digest.py`.
- Add: New function `write_{mode_name}(verdicts, period, **kwargs)` → returns `(filename, markdown_text)`.
- Register: Add entry to `writers` dict in `main()` (line 346).
- Test: Add test case to `tests/test_write_bluf.py` (or new file `test_write_{mode_name}.py`).

**Extend Scorer Enrichment (New Field):**
- File: Edit `score/triage_score.py`.
- In `score_item()` prompt: Add new field to the JSON enum and worked examples.
- In response parsing: Handle the new field in the fallback dict assignment (lines 139–140).
- In digest: Update `write_brief()` etc. to use the new field if relevant.
- **Sync:** If the new field is a CCIR tier or ordering dependency, also update `score/digest.py:CCIR_ORDER` and ensure drift guard is synchronized.

**New Test:**
- File: `tests/test_{feature}.py`.
- Pattern: `import pytest`, define fixtures, write `test_` functions.
- Run: `python3 -m pytest tests/test_{feature}.py -v`.
- Coverage: Test both happy path and error cases.

**Utilities:**
- Shared module: `score/_util.py` (imported by digest, fever_triage).
- Example: `bridge/_util.py` has `escape()` (HTML entity encoding for Atom).
